User Roles and Permissions

PharmaLogic uses a role-based access control system to ensure data security and efficient workflows.

  • Admin: Full access to all features, including user management and system settings.
  • Manager: Access to reporting, inventory, and user assignment tools.
  • Staff: Limited access to patient records and day-to-day task management.
  • Viewer: Read-only access to data for auditing purposes.

Roles can be customized based on your organization's needs through the user-management module:

pharmalogic-user --role customize

FAQs and Troubleshooting

Common Issues and Solutions

Q: How do I reset my password?

A: To reset your password, click on the Forgot Password link on the login page. You'll be prompted to enter your email address, and a password reset link will be sent to your inbox.

Q: Why am I getting a 500 server error?

A: A 500 server error indicates an issue with the PharmaLogic server. Please check our status page for updates. If the issue persists, contact support at support@pharmalogic.com.

Q: How can I import patient data into PharmaLogic?

A: PharmaLogic supports importing data in CSV format. To upload patient records, navigate to the Data Import section in the admin panel, select the CSV file, and follow the on-screen instructions.

Q: How can I integrate PharmaLogic with my existing EMR system?

A: You can use the PharmaLogic API to integrate with your existing Electronic Medical Records (EMR) system. Refer to the API and Integration section for detailed documentation on the available endpoints and authentication methods.

Q: How do I update the software?

A: To update PharmaLogic to the latest version, navigate to the System Settings page and click on the Check for Updates button. If an update is available, follow the on-screen instructions to install it.

API and Integration

PharmaLogic offers an extensive API that facilitates seamless integration with external systems, enhancing its flexibility and scalability. The API is designed to be RESTful, providing you with simple and efficient methods to interact with the data in the PharmaLogic platform.

API Endpoints

Below are the key API endpoints available:

  • /api/v1/patients/{id}: Fetches detailed patient information by ID.
  • /api/v1/inventory: Retrieves current inventory data.
  • /api/v1/reports: Generates customizable reports based on specific filters.
  • /api/v1/medications/{id}: Retrieves information about specific medications.

Example Usage

Example API call to fetch patient data:

GET /api/v1/patients/{patientId}
Authorization: Bearer 
Response:
{
    "id": 12345,
    "name": "Jane Doe",
    "age": 29,
    "medications": ["Ibuprofen", "Amlodipine"],
    "medicalHistory": ["Allergic to penicillin"]
}

API Authentication

All API requests require authentication using a Bearer token. You can obtain a token by logging into your PharmaLogic account and using the /auth/token endpoint to generate a new token.

POST /api/v1/auth/token
Request Body:
{
    "username": "admin",
    "password": "securepassword"
}
Response:
{
    "access_token": ""
}
